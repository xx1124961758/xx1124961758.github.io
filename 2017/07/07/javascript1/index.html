你应该知道的JavaScript的 4 种设计模式！

发表于 2017-07-07 | 分类于 JavaScript | 阅读次数

每一个开发者都在努力写出具有可维护性，可读性，可复用的代码。随着应用规模的变大，代码的结构化变得更加重要。设计模式 （ 在一种在特定场景下，为普遍的问题提供一个组织结构 ） 是解决这一挑战的关键。

JavaScript web 开发者在创建应用的时候，经常会用到设计模式 （ 即使他们没有意识到 ）。

虽然在某些情况下使用到的设计模式很多样化，但是 JavaScript 开发者通常比其他人更倾向使用一些设计模式。

在这篇文章中，我想讨论下一些普遍的设计模式，以提高你们的编程能力和深入 JavaScript 内部。

将要讨论的设计模式有：

  • 模块（Module）
  • 原型（Prototype）
  • 观察者（Observer）
  • 单例（Singleton）

每一种设计模式都包含很多属性，不过，我会着重强调下面几个关键点：

  1. 上下文（Context）: 在什么情况下使用或者在哪里使用？
  2. 问题（Problem）: 我们会尝试解决什么问题？
  3. 解决方法（Solution）: 用这个设计模式解决我们提出的问题怎么样？
  4. 实现（Implementation）: 实现起来是什么样的？

”模块“设计模式

JavaScript 模块是普遍用来保持特定的代码片段独立于其他组件的设计模式。这提供了松散耦合，结构良好的代码。

对于那些熟悉面向对象编程语言的开发者，模块是 JavaScript 的 “class”。类的优点之一就是 “封装” - 能够保护那些被其他类访问的属性和方法。模块设计模式运行定义 “公共的” 或者 “私有的” 不同的级别。

原型设计模式

任何一个 JavaScript 开发者看到过这个 prototype 关键字。被原型继承或者在他们的代码中实现继承所困扰过。原型设计模式依赖于 JavaScript 的原型继承。原型模式大多数被用来高性能地创建对象。

被源对象浅复制创建的对象到处传递。原型模式的一个使用场景就是执行一个广泛的数据库操作，创建一个对象以供应用的其他部分所使用的。如果另一个进程需要使用这个对象，可以使用之前创建的对象，而不是重复大量数据库的操作。
